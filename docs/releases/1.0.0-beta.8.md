## Release scope

Beta 8 is a focused Android Release/AOT stability release. It retains the Beta
7 workflow and data model while fixing two crashes reached during automatic
online vocabulary preparation:

1. reflection-dependent JSON serialization in persisted lookup and diagnostic
   paths; and
2. an AOT-unsafe AngleSharp CSS-selector path in the Wiktionary parser.

There are no intentional user-workflow or database-schema changes in this
release.

## AAB creation

The released artifact was a signed Android App Bundle built from the release
commit for `net10.0-android` in `Release`.

The checked-in Release configuration enables:

- optimization;
- trimming;
- Android AOT compilation; and
- embedded managed assemblies.

The parameterized Google Play publisher supplies
`AndroidPackageFormats=aab`, the version code, display version, production
package ID, and the external signing identity. Signing material remains outside
the repository. The generated AAB is ignored and is not committed.

The post-release brief confirms successful AAB creation and signing. The
artifact SHA-256 was not preserved in the repository, so this document does not
invent one.

## AOT and trimming changes

`LexicalJsonSerializerContext` supplies source-generated metadata for
`LexicalResult` and string aliases. The affected cache, preparation,
learning, review-diagnostics, and logging paths use explicit generated type
information instead of reflection-dependent generic serialization.

The Wiktionary parser replaces the affected selector-matching operations with
explicit DOM traversal plus class and attribute checks. The release includes a
dedicated HTML fixture and parser regression tests.

## Fixed crashes

- Android Release no longer terminates while persisting a successful automatic
  online lookup because the required JSON metadata is unavailable under AOT.
- The Wiktionary parser no longer enters the AOT-unsafe CSS-selector path
  reached by the tested lookup response.
- Provider failures are contained and presented as controlled preparation
  outcomes instead of terminating the process.

## Data compatibility

- Database schema version remains **7**.
- Beta 8 adds no tables, columns, or migrations relative to Beta 7.
- Existing personal learning state uses the same schema and lifecycle rules.
- Legacy lexical-cache-key invalidation behavior is unchanged.
- The automated migration fixture uses an isolated temporary database and
  remains green.

No versioned backup or restore format exists. Users must not be told that
copying a live SQLite file is a supported backup procedure.

## Known limitations

- No supported backup, restore, export, or cloud synchronization.
- Cache misses require network access and explicit Wikimedia consent.
- iOS and Mac Catalyst are not release-validated product platforms.
- The repository does not retain the released AAB, its checksum, physical
  device details, or completed screenshots.
- The direct-install Android test-package script still contains legacy Beta 6
  labels and is not the Beta 8 release path.
